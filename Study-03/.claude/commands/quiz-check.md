모든 퀴즈 문제의 정답 정확성을 검증합니다.

$ARGUMENTS 값이 있으면 해당 카테고리(bible, hiphop, nba, mlb)의 파일만 검증하고,
값이 없으면 `quiz-game/src/data/` 디렉터리의 bible.ts, hiphop.ts, nba.ts, mlb.ts 전체를 검증합니다.

## 진행 절차

### 1단계: 대상 파일 읽기
해당 카테고리의 데이터 파일을 읽어 모든 문제를 파싱합니다.
각 문제에서 `question`, `options`, `answer`, `explanation` 필드를 추출합니다.

### 2단계: 문제별 정답 검증

각 문제에 대해 아래 항목을 확인합니다.

1. **정답 인덱스 유효성**: `answer` 값이 0~3 범위인가?
2. **정답 사실 확인**: `options[answer]`에 해당하는 선택지가 사실로서 정확한가?
   - 수치/통계 문제: 공식 기록과 일치하는지 확인
   - 인물/사건 문제: 역사적 사실과 일치하는지 확인
   - 의심스러운 경우 WebSearch를 사용하여 교차 검증
3. **해설 정합성**: `explanation`의 내용이 정답과 모순되지 않는가?
4. **오답 배제 확인**: 나머지 3개 선택지가 정답이 될 가능성은 없는가?
5. **문제-정답 일치**: 문제가 묻는 내용과 정답이 정확히 대응하는가?

### 3단계: 검증 결과 분류

각 문제를 아래 상태로 분류합니다.
- **정확**: 정답이 사실과 일치하고 해설도 정확함
- **의심**: 정답이 맞을 가능성이 높으나 확인이 필요한 부분이 있음
- **오류**: 정답이 사실과 불일치하거나 해설에 오류가 있음

### 4단계: 오류 발견 시 수정안 제시

오류로 분류된 문제에 대해 아래를 제시합니다.
- 현재 정답과 올바른 정답
- 수정된 `answer` 값 (인덱스가 변경되어야 하는 경우)
- 수정된 `explanation` 텍스트
- 선택지 자체를 수정해야 하는 경우 수정된 `options`

**수정은 자동 적용하지 않고, 제안만 합니다.** 사용자 확인 후 적용합니다.

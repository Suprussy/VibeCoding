카테고리 인수가 주어지면 해당 카테고리의 문제 전체를 새 문제 10개로 교체합니다.

- $1 (필수): 교체할 카테고리 (bible, hiphop, nba, mlb)

---

## 진행 절차

### 0단계: 대상 카테고리 결정

- `$1`이 없는 경우: 아래 메시지를 출력하고 **즉시 중단**합니다.

  ```
  ❌ 카테고리를 지정해주세요.
  사용법: /quiz-change-all <카테고리>
  카테고리: bible | hiphop | nba | mlb

  한 번에 하나씩 실행해야 출력 한계를 피할 수 있습니다.
  ```

- 유효하지 않은 카테고리명이 입력된 경우도 오류를 알리고 중단합니다.
- 유효한 카테고리가 주어진 경우에만 다음 단계로 진행합니다.

---

### 1단계: 기존 파일 백업

1. `quiz-game/src/data/{카테고리}.ts` 파일을 읽습니다.
2. 오늘 날짜(YYYYMMDD)를 이용해 백업 파일을 생성합니다.
   - 경로: `quiz-game/src/data/backup/{카테고리}.{날짜}.bak.ts`
   - 예: `quiz-game/src/data/backup/bible.20260628.bak.ts`
   - 이미 같은 날짜의 백업이 있으면 `.bak2.ts`, `.bak3.ts` 등으로 구분합니다.
3. 백업이 완료된 후에만 다음 단계로 진행합니다.

---

### 2단계: 새 문제 10개 작성

기존 백업 파일의 문제와 주제·정답이 겹치지 않는 **완전히 새로운** 문제 10개를 작성합니다.

#### 공통 품질 기준

아래 6가지를 모두 충족해야 합니다.

1. **정답 유일성**: 정답이 하나뿐인가? 다른 해석이 가능하면 조건을 명시합니다.
2. **최상급 표현 기준**: '가장 큰', '최초의', '역대' 등의 표현에 측정 기준(범위, 시점, 리그/분류)이 명시되어 있는가?
3. **시간·범위 명확성**: 변할 수 있는 정보에는 시점을 명시합니다. 지리적·분류적 범위를 한정합니다.
4. **교차 검증**: 정답 사실이 정확한지 확인합니다. 의심스러운 정보는 2개 이상 출처 기준으로 판단합니다.
5. **기존 문제 중복 없음**: 같은 파일(백업 포함) 내 유사한 주제의 문제가 없어야 합니다.
6. **오답 선택지 그럴듯함**: 오답이 너무 명백하게 틀리지 않아야 합니다. 실제로 헷갈릴 수 있는 값으로 구성합니다.

#### 정답 분포 규칙

10문제의 `answer` 값(0, 1, 2, 3)이 가능한 한 고르게 분포해야 합니다.
- 이상적 분포: 0번 2~3개, 1번 2~3개, 2번 2~3개, 3번 2~3개
- 특정 번호가 4개 이상 몰리지 않도록 합니다.

#### 카테고리별 출제 방향

| 카테고리 | 출제 포인트 |
|---------|------------|
| bible   | 구약/신약 균형, 구체적 수치·인물·사건 (팔린 가격, 재위 기간, 규빗 수 등) |
| hiphop  | 90~2010년대 중심, 앨범명·피처링·레이블·수상·역사적 사건 |
| nba     | 역대 기록·통계·MVP·파이널·드래프트 수치 기반 |
| mlb     | 역대 기록·월드시리즈·전설적 선수 수치·수상 기반 |

#### 난이도 기준

- 마니아/전문가 수준 (단순 상식 X)
- 구체적 수치, 연도, 통계, 이름 등 암기 없이는 풀기 어려운 문제

### 3단계: 파일 전체 교체

아래 형식으로 해당 파일을 완전히 새로 작성합니다.
id는 `{카테고리}_001`부터 `{카테고리}_010`까지 순서대로 부여합니다.

```typescript
import { Question } from '@/types/quiz'

const {카테고리}Questions: Question[] = [
  {
    id: '{카테고리}_001',
    category: '{카테고리}',
    question: '문제 텍스트',
    options: ['선택지1', '선택지2', '선택지3', '선택지4'],
    answer: N,
    explanation: '해설 텍스트 (출처 포함)',
  },
  // ... 총 10개
]

export default {카테고리}Questions
```
